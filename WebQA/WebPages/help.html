Introduction

WebQA is a web based program that allows to search the requirements converted from MS Excel (.xls) files which are created in special format.




Features

There are short list of features of the WebQA:

  • Search requirement(s) by FR ID, FR TMS Task or FR text;
  • Searching by FR ID diapason;
  • Multi-word, -id, -tms task searching;
  • Share search results with colleagues;
  • Filtering by FR source.



Search requirements

Search criteria section contain following options:

  • "Search by" drop down list

    • Select an option to specify what data will be used for qualifying requirements.
  • "Value to search" field

    • Define the search keyword or keywords into this field which will be used for requirements searching. This can be requirement number(s), tms task number(s) or some word(s).
  • "Search" button

    • Press "Search" button to perform searching.

WebQA allows to search requirements by FR ID, FR TMS Task and FR text. You should specify search criteria by selecting appropriate option from the "Search by" list and then define search keyword(s). All searching is case-insensitive.

Search keywords can be separated by comma or semicolon. In this case found requirements will contain all specified keywords (AND logic). There is one more option available and can be used for searching groups of requirements by FR ID only. For this purpose specify two numbers separated by dash. This lind of search is called as "FR ID diapason" and has limitation by 5000 numbers.

There are examples of search keywords:
Search by Value to search Description
FR ID FR1 search for requirement with number 'FR1'
FR ID FR1;fr3 search for requirements with numbers 'FR1' or 'FR3'
FR ID FR1,fr3 search for requirements with numbers 'FR1' or 'FR3'
FR ID FR1-FR3 search for requirements with numbers 'FR1' or 'FR2' or 'FR3'
FR ID FR1-3 search for requirements with numbers 'FR1' or 'FR2' or 'FR3'
FR ID NFR1-3 search for requirements with numbers 'NFR1' or 'NFR2' or 'NFR3'
FR TMS Task PCC-43345 search for requirements which contain in FR TMS Task 'PCC-43345'
FR TMS Task PCC;mar5 search for requirements which contain in FR TMS Task 'PCC' and 'MAR5'
FR TMS Task PCC,mar5 search for requirements which contain in FR TMS Task 'PCC' and 'MAR5'
FR Text OE_param_micro search for requirements which contain in text specified keyword
FR Text oe;specimen;collect;receive search for requirements which contain in text all the keywords specified
FR Text oe,specimen,collect,receive search for requirements which contain in text all the keywords specified

When keyword is specified you need to press Enter or click "Search" to perform searching.

Note that you can use up and down arrows on the keyboard to switch between "Search by" options when the focus is in "Value to search" field.

There is one more thing you should know. All searches are limited by search results by default. This means that WebQA will break searching in case more than 100 requirements are found. If you see Only first 100 results are displayed! message then specify more clear search keywords and repeat searching.




Columns description

  • FR ID - requirement identifier;
  • TMS Task - TMS task number associated with requirement;
  • Object # - "Object number", interaction level of the requirement;
  • Text - Text of the requirement;
  • CCP - Level of the failure:
    • 5 - Critical, leading to serious permanent injury or death;
    • 4 - Critical, could cause serious delay or misinformation;
    • 3 - Failure that could cause to extended delay in patient care;
    • 2 - Failure that could cause to delay in treatment of patient;
    • 1 - Minor nuisance to users.
  • Created / Modified - Dates when requirement was created and last modified;
  • Is changed? - Are created/modified dates are equals or not?
  • Status - Status of the requirement
  • Found in file - Source name where from requirement was imported to WebQA system.
