Introduction

jRQS is a free open-source web application that allows to make search for the requirements parsed from Excel spreadsheets.

You should always keep in mind that search results might be outdated since jRQS search for the requirements parsed from Excel spreadsheets. The search results actuality depends on when jRQS's database was last updated by the jRQS instance administrator.


How to search

Generally, it is possible to search for the requirements by exact FR ID, FR TMS task coincidence and by the keyword or a bunch of keywords separated by comma. In addition, it is possible to search by exact rowid in the database and filter out the results by FR sources separated by comma.

All the search is case-insensitive.

There are few examples indicated below to help get you involved:

Search by Example Description
FR ID FR1 Search for the concrete requirement by it's FR ID
FR1, FR2, FR3 Search for a bunch of requirements by their FR IDs
FR1-3 Search for a bunch of requirements by range of FR IDs.
In addition, search by range may be combined with standard search requests of the same type e.g. FR1,FR3-5,NFR6-10.
Please note that range size is limited to 100, so the search request like FR100-201 will be treated as invalid. Moreover, overall search parameters are limited to the 100, so search requests containing more than maximum allowed parameters will be treated as invalid as well.
TMS task COBRA-12345 Search for requirements associated with the TMS task specified.
Please note that jRQS will look not for the concrete TMS task but for the tasks that looks like as specified one. For example, search by COBRA-1234 will find the COBRA-12345 as well.
COBRA-12345, COOKIE-12345 Search for requirements by several TMS tasks
Keyword keyword Search for requirements by the keyword
keyword1, keyword2 Search for requirements by several keywords.
Please note that AND logic is applied for such cases, so all requirements containing both keywords will be found.

Search results are limited to 100. You will be notified by the message if your search request returned 100 or more results.

Filter out the results

If you got big search results, you may click on the source link to repeat the search using the only one particular source to make your search results more clear. Furthermore, you may specify more than one source to filter out the search results by separating them with comma, but this should be done manually since UI does not provide such possibility. The same works for the search by rowid.


Grid columns

Column Description
Actions Contains the action button to share the link to a particular requirement
FR ID Requirement's unique identifier per project
TMS Task TMS task number associated with the requirement
Object Number Interaction level of the requirement
Text Requirement's text itself
CCP Level of the failure, where:
 5 is a critical, leading to serious permanent injury or death,
 4 is critical, could cause serious delay or misinformation,
 3 is a failure that could cause to extended delay in patient care,
 2 is a failure that could cause to delay in treatment of patient,
 1 is a minor nuisance to users
Created / Modified The dates when requirement was created and modified last time. In addition, the indicator appears in case created and modified dates are not equal
Status Requirement's creation/review status
Boundary Indicates is boundary testing should be performed for checking the requirement or not
Source The source file name where from the requirement was imported into the jRQS database
